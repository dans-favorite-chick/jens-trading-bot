### Scope reminder

Tonight's commits all land on branch `feature/knowledge-injection-systems`. The tag `v-pre-impl-2026-04-19` marks the branch tip BEFORE tonight's work. Rollback = move branch tip back to the tag.

### Option A — soft reset (SAFEST: preserves files on disk, un-commits changes)

```bash
cd "C:/Trading Project/phoenix_bot"
git reset --soft v-pre-impl-2026-04-19
git status                 # all tonight's changes now staged, ready to review
git diff --cached --stat   # what tonight touched
# Now you can:
#   - unstage + modify: git reset HEAD <file>
#   - discard selectively: git checkout -- <file>
#   - re-commit in smaller pieces: git add <file> && git commit -m "..."
```

### Option B — nuke a single bad commit without touching others (RECOMMENDED for mid-sprint recovery)

```bash
git log --oneline v-pre-impl-2026-04-19..HEAD   # see commits since tag
git revert <bad-sha>                             # creates an un-do commit (non-destructive)
# The bad commit stays in history; the revert commit undoes it.
```

### Option C — move branch back, keep the work on a side branch (RECOMMENDED for full-sprint abort)

```bash
git branch feature/knowledge-injection-systems-attempt-1 HEAD
git reset --hard v-pre-impl-2026-04-19
# The dead-end work is fully preserved on the new branch name.
# Current branch is back at the safety tag. Can start fresh cleanly.
```

### Option D — hard reset (DESTRUCTIVE: discards commits AND working-tree changes)

```bash
# WARNING: this deletes uncommitted work in the working tree.
# Only use if you're certain everything you want is already committed somewhere.
git reset --hard v-pre-impl-2026-04-19
```

### Recovery if a reset goes wrong

```bash
git reflog | head -20         # shows every HEAD movement; find the lost SHA
git reset --hard <that-sha>   # restore
```
